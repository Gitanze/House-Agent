# 房源内容矩阵口播

## 工作流程

1. 先整理房源事实：面积、户型、空间组成、连接关系、人工复核、逐房实景事实和人工补充亮点。
2. 读取 `references/voice-rules.md`，应用通用语言、断句和事实边界。
3. 读取 `references/audience-rules.md`，把用户输入的目标客户转成实际关注点。
4. 读取 `references/voice-types.md`，按所选口吻建立叙事身份。
5. 读取 `references/focus-types.md`，按所选重点决定结构。
6. 读取 `references/matrix-composition.md`，把目标客户、口吻、重点和人工亮点合成脚本策略。
7. 如存在 `cases/library.json`，只读取与当前口吻和重点匹配的案例，学习节奏与表达强度，不复用案例房源事实。
8. 输出完整口播正文；不要解释创作过程。

## 必守原则

- 人工补充亮点必须进入口播，并成为更细的展开方向。
- 房源事实只写已确认或可从资料中直接得出的内容。
- 装修相关的内容不可进行无凭据拓展，例如全屋智能的能力范围等不确定的就不写。
- 抽象口吻可以极致、好笑、夸张，但不能虚构不存在的房间、学校、改造、配套或居住事实。
- 不说房子的不好；如需表达选择感，改写成正向生活取舍。
- 每条口播围绕一个核心命题，不写成卖点清单。
