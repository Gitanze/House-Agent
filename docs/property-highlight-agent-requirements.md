# 小红书房源亮点策划 Agent 需求文档

> 当前 MVP 已切换为“本地手工案例库模式”：读取
> `rednote/content_data/samples.json`，不调用 rednote 爬虫，也不再进行关键词确认。

## 1. 背景与目标

系统最终将演化为多个 Agent 协作。现阶段已经有平面图识别 Agent，下一步新增独立的“小红书房源亮点策划 Agent”。

两个 Agent 的职责必须清晰分离：

- 平面图识别 Agent：只负责客观识别户型结构、房间关系和空间特征。
- 房源亮点策划 Agent：结合客观户型结果、人工房源资料和小红书热门内容，生成亮点策略与 30 秒亮点口播。

## 2. 使用边界

- 当前为研究、学习和演示用途。
- 小红书采集使用仓库内已经跑通的 `rednote` 工具。
- rednote 仅用于个人学习、研究和演示，不进行大规模采集。
- 未来商业化时，采集 Provider 必须替换为获得授权的数据 API。
- 不抓取评论，不下载图片或视频，不自动发布内容。
- 只提炼热门内容的钩子、结构、关键词和节奏，不复制或仿写大段原文。

## 3. 输入

### 3.1 客观户型分析

接收第一个 Agent 的 `floorplan-analysis/v1` 输出，至少包含：

- 户型、面积、朝向
- 房间和空间连接关系
- 采光、通风、动静分区、餐厨动线等空间特征
- 不确定项和人工复核项

### 3.2 结构化房源信息

- 城市
- 板块或区域
- 小区名

### 3.3 人工补充亮点

允许逐条输入平面图无法判断的信息，例如：

- 装修交付情况
- 景观、楼层体验
- 学区、交通
- 税费或产权信息

人工信息必须在输出中标记来源，不得冒充识图结论。

## 4. LangGraph 工作流

```text
整理房源上下文
→ 自动生成 3–5 组检索关键词
→ 暂停并让用户编辑确认
→ 查询 7 天本地缓存
→ rednote 采集缺失关键词
→ 数据清洗与笔记去重
→ 互动热度评分
→ 房源相关性筛选
→ 提炼标题钩子、叙事结构和关键词
→ 生成房源亮点策略
→ 生成 30 秒亮点口播
→ 事实一致性审核
→ 输出
```

## 5. 小红书数据获取

- rednote 作为独立 Flask 服务运行，Agent 通过 `RednoteProvider` 调用。
- rednote 使用 Playwright 持久化浏览器目录和 Cookie 保存登录态。
- 首次登录或 Cookie 失效时由用户在 rednote 打开的浏览器中完成登录。
- 相同关键词优先使用 7 天缓存；缺少或过期时才触发采集。
- 每组关键词最多取约 30 条，按笔记 ID 去重，最终筛选 5–10 篇。

## 6. 排序与筛选

### 6.1 互动热度

在缺少曝光量或粉丝数时，不将结果称为真实互动率。

互动热度综合：

- 收藏数，最高权重
- 评论数
- 点赞数
- 分享数
- 发布时间衰减

互动数采用对数归一化，减少超大账号和历史老内容的天然优势。若数据源能够提供作者粉丝数，再补充真实互动率。

### 6.2 内容相关性

第二层筛选只保留符合当前需求的笔记：

- 房产、户型、买房或居住体验相关
- 与目标城市、板块、小区或相似户型有关
- 能支持当前房源亮点表达

过滤：

- 装修教程但与房源销售无关
- 招商、中介招募和纯广告
- 无关城市或无关房型
- 互动高但无法帮助当前房源表达的内容

## 7. 输出协议

版本：`property-highlight-plan/v1`

输出至少包括：

- `searchQueries`：确认后的搜索关键词
- `sourceNotes`：入选笔记、来源链接、互动指标和筛选理由
- `trendPatterns`：热门钩子、内容结构、关键词和表达节奏
- `highlightStrategy`：房源核心亮点排序、目标客群和事实依据
- `talk30s`：真正聚焦 2–3 个亮点的 30 秒口播
- `warnings`：来源不足、事实风险和待人工确认项
- `runMetadata`：Provider、缓存命中和执行路径

## 8. 30 秒口播要求

- 前 3 秒直接给出最强亮点。
- 只讲 2–3 个最有依据的亮点。
- 每个亮点说明对真实居住生活的价值。
- 人工亮点优先，但明确标记为用户提供。
- 不流水账介绍几室几厅。
- 不复现参考笔记原文。
- 风险提示独立输出，不占用核心口播。

## 9. 前端流程

1. 完成客观户型分析。
2. 填写城市、板块、小区和人工亮点。
3. 启动房源亮点策划 Agent。
4. 查看并编辑 Agent 生成的 3–5 组关键词。
5. 确认后开始采集和筛选。
6. 查看参考趋势、亮点策略和 30 秒口播。

## 10. 验收标准

- 两个 Agent 使用独立目录、Schema、LangGraph 和 checkpoint namespace。
- 第一个 Agent 不再负责热门内容检索和 30 秒亮点口播。
- rednote 不可用时返回明确状态；缓存可用时仍能继续。
- 无关爆款即使互动高也会被过滤。
- 每个最终亮点都能追溯到户型事实、人工资料或参考趋势。
- 自动测试覆盖关键词确认、缓存、采集失败、去重、评分、相关性筛选和事实审核。
