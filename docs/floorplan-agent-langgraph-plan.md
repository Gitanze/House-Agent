# 平面图识别 LangGraph Agent 实施计划

## 总体目标

将现有“户型图识别并产出亮点”能力改造成可独立复用的 LangGraph Agent，作为未来多 Agent 系统的第一个上游 Agent。

本 Agent 只负责：

- 户型客观识别
- 空间特征判断
- 亮点、短板、适合人群
- 不确定项与人工复核

30 秒讲解词不属于该 Agent，后续由文案 Agent 负责。

## 实施步骤

### 1. 建立最小 LangGraph（已完成）

- 引入 `@langchain/langgraph` 和 Zod。
- 定义统一的 `FloorplanAgentState`。
- 将现有视觉模型调用封装为 `recognizeFloorplan` 节点。
- 建立 `START → recognize → END` 最小图。
- 保持现有 API 和前端可运行。

### 2. 用 Schema 约束输出（已完成）

- 用 Zod 定义房间、空间关系、专业特征及完整识别结果。
- 视觉节点只输出客观识别信息。
- 增加校验节点，检查枚举、房间 ID、连接引用及字段完整性。

### 3. 条件路由与修复（已完成）

- 增加结构修复节点。
- 校验通过时继续，能修复时最多修复一次，无法可靠修复时进入人工复核。
- 不允许修复节点凭空补充房间、面积或朝向。

### 4. 分离空间推理与亮点生成（已完成）

- 独立分析采光、通风、动静分区和厨餐动线。
- 独立生成 `pros`、`cons`、`suitableFor` 及判断依据。
- 亮点节点只读取结构化识别结果，不直接读取原图。

### 5. 事实一致性复核（已完成）

- 检查每个亮点是否有识别事实支撑。
- 对朝向、通风、采光和尺度判断增加确定性规则。
- 输出警告、人工复核项和审核结论。

### 6. SQLite Checkpoint 与人工复核（已完成）

- 使用 SQLite 保存 LangGraph checkpoint。
- 每次执行使用稳定 `threadId`。
- 通过 `interrupt()` 暂停流程，人工编辑或批准后用相同线程恢复。

### 7. 稳定接口与测试（已完成）

- 定义 `floorplan-analysis/v1` 输出。
- 保持现有分析 API，内部切换到 Graph。
- 增加人工审核恢复 API。
- 增加 Schema、节点、路由和完整图集成测试。

## 教学与交付方式

- 每一步都形成可运行、可验证的检查点。
- 每完成一步，解释对应 LangGraph 原理和设计原因。
- 用户验收当前步骤后，再进入下一步。

## 技术约定

- 沿用 Node.js/TypeScript 技术栈。
- 使用 LangGraph JS。
- 第一阶段不接入 LangSmith。
- 不扩展或重构 benchmark 范围。
- 保留现有未提交工作，避免覆盖用户修改。
